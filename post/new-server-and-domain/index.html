0|hugo server

想直接使用hugo server来测试，然而配置了一番，端口啥的都弄了，服务器上跑的hugo server没法从域名访问到。

1|caddyserver

还是用caddy跑个页面测试吧。

跟随 https://caddyserver.com/docs/install#debian-ubuntu-raspbian 文档操作安装caddy

然后新建了一个测试目录，进入目录，新建一个文件Caddyfile，内容如下：

:80
respond "Hello, world!"

然后在目录下运行caddy adapt，成功运行起来。

再通过tecnico.cc访问，显示出了Hello, world!的字样。测试成功。

由于监听80, 443等端口需要更高的权限，所以需要使用sudo权限启动caddy

先caddy stop，再sudo caddy start即可。

3|pages

未备案不能用域名访问，但是直接用ip地址访问来测试还是没问题的。

新建一个目录~/www/test，然后把测试的网页复制进去。

修改Caddyfile的内容为：

:80

root * /home/ubuntu/www/test
file_server

然后caddy reload更新配置。刷新网页即可看到页面正常显示出来了。

Caddyfile的root命令不支持~目录，需要写入完整的路径

或者可以使用环境变量写作root * {$HOME}/www/test

更多caddyfile用法参见官方文档
